## 第1页 课程导览：机器感知在做什么

### 一句话定义
**机器感知 = 机器人通过传感器"看见"自己 + "看见"世界，并据此决策与运动。**

### 机器感知的核心矛盾
- **机器人位置不确知**：里程计、IMU 都有累积误差
- **环境地图未知**：第一次进房间，没有先验地图
- **目标位姿不确知**：要抓的杯子在哪儿、朝哪儿

> 这三个"不知道"互相耦合，导致鸡生蛋蛋生鸡 → **SLAM 问题**

### 课程主线
```
前端 VO（视觉里程计）     → 局部连续位姿
  ├─ 2D-2D：对极几何
  ├─ 3D-2D：PnP / BA
  ├─ 3D-3D：ICP
  └─ 光流 / 直接法
后端优化（GraphSLAM）     → 全局一致位姿
  └─ 回环检测消除累积漂移
```

## 第2页 机器感知的目的与任务全景

### 机器人需要回答的六个问题

| 任务 | 含义 | 输入 | 输出 |
|---|---|---|---|
| **自身定位** | 我在哪 | 传感器观测 | 位置 [x,y,z] + 姿态 [pitch,roll,yaw] |

| **目标检测** | 眼前有什么 | 图像 / 点云 | 类别 + 2D/3D 框 |

### 任务层次
```
基础版：哪里有障碍物？
```

## 第3页 五大技术手段：视觉·惯导·雷达·轮式·GNSS

### 视觉模组

| 类型 | 优点 | 缺点 |
|---|---|---|
| **单目** | 成本低、体积小 | 丢失深度 |
| **双目** | 视差算深度 | 强光暗光受限，精度有限 |
| **RGB-D** | 物理手段直接得深度 | 室内为主，户外难 |
| **结构光** | 弱光好 | 远距离差 |
| **ToF** | 测距远（百米） | 近距精度差 |

### 惯性测量单元 IMU
- **三轴陀螺仪** → 测角速度（方向/翻滚）
- **三轴加速度计** → 测加速度
- **磁力计** → 协助校准方向漂移

陀螺仪分类：
- **机械陀螺仪**（萨格纳克效应 1913）→ 高精度昂贵
- **激光陀螺仪 GG1320AN** → 88×88×45mm, 1.6W, 漂移 0.0035°/h，价格 >$5000
- **MEMS 陀螺仪 MPU6050** → 5×5×5mm, 18mW, 漂移 <70°/h，价格 $20

> MEMS 在 200Hz 采样下 10 分钟内漂移严重，需要外部修正。

### 激光雷达 LiDAR
- 机械式 / 纯固态 / 混合固态
- 雷达 SLAM 与视觉 SLAM 流程基本相同

### 轮式编码器
- 通过光电原理计算圈数和角度
- 已知轮距 + 半径 → 推算位置和姿态

### GNSS 全球导航卫星系统
- GPS / 北斗 / GLONASS / Galileo
- 普通 GNSS 误差 ~10m
- **RTK**（实时差分）→ 厘米级，需基准站 + 流动站

## 第4页 SLAM 问题的形式化与难点

### 形式化定义

给定智能体在 `1..T` 时刻的**观测信息** `{o₁,...,o_T}` 与**运动控制信息** `{u₁,...,u_T}`，SLAM 的目标是计算机器人 `1..T` 时刻的运动轨迹 `{p₁,...,p_T}` 与环境地图 `M`：

$$
\arg\max_{p_{1:T},M} \; P(p_{1:T}, M \mid o_{1:T}, u_{1:T})
$$

### 为什么 SLAM 是难问题？

1. **轨迹与地图都未知** → 鸡生蛋问题
2. **路径误差与地图误差相互耦合**
3. **数据关联未知**：当前检测的特征点 ↔ 地图中哪个特征点？错配就大错
4. **累积漂移**：纯前端里程计必然漂

### 三个解决手段

```
1. 感知冗余 + RANSAC  →  鲁棒匹配
2. 后端图优化          →  全局一致性
3. 回环检测            →  远距离约束，纠正累积误差
```

## 第5页 3D 位姿变换基础：点·向量·坐标系

### 基本元素
- **点**：三维空间中的位置
- **向量**：带方向箭头，可加减、数乘
- **坐标系**：三个正交轴构成的基（左/右手系）

### 内外积
- **内积** `a·b = Σaᵢbᵢ` → 标量，描述相似性
- **外积（叉积）** `a×b` → 矢量，正交于 a,b 张成的平面

### 2D 旋转变换（向两种理解）

**主动旋转**：向量 p 旋转 θ
$$
p' = R(\theta)\, p
$$

**被动旋转**：坐标系旋转 θ（向量在新坐标系下的坐标）

> 核心结论：**向量旋转 θ ≡ 坐标系旋转 -θ**

### SLAM 中的核心符号

| 符号 | 含义 |
|---|---|
| `T_wb` | 世界到机体的变换（机体的位姿） |
| `T_bw` | 机体到世界的变换 |
| `p_w` | 点在世界坐标下的坐标 |
| `p_b` | 点在机体坐标系下的坐标 |
| 关系 | `p_w = T_wb · p_b` |

机器人感知世界 = 1) 感知 `p_b` + 2) 追踪 `T_wb`

## 第6页 旋转矩阵与 SO(3) 群

### 定义
$$
R(\theta) = \begin{bmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{bmatrix}
$$

### 性质
1. **正交**：R Rᵀ = I
2. **行列式 = 1**：det(R) = 1
3. **R⁻¹ = Rᵀ**

### 特殊正交群 SO(n)
$$
SO(n) = \{ R \in \mathbb{R}^{n\times n} \mid RR^T = I, \det(R) = 1 \}
$$

满足群的四个公理：**封闭性 / 结合律 / 单位元 / 逆元**。

### 3D 旋转矩阵的构造
= 绕 x, y, z 三轴旋转的合成（顺序不可换！）

### 连续旋转的乘法顺序
- **动轴**（载体坐标系，SLAM 常用）→ **右乘**
- **静轴**（世界坐标系）→ **左乘**

> SLAM 中追踪 `T_wb` 时载体坐标系是动轴 → 旋转矩阵右乘

### 特殊欧氏群 SE(3)
$$
T = \begin{bmatrix} R & t \\ 0^T & 1 \end{bmatrix} \in SE(3)
$$
齐次坐标下，平移和旋转可以用一次矩阵乘法表示，连续变换非常简洁。

## 第7页 欧拉角·角轴·四元数三种姿态表达

### 欧拉角
- 用三个角度 `α,β,γ` 描述姿态
- **本质：描述"过程"（绕轴旋转了多少），不是"状态"**
- 12 种约定（Proper + Tait-Bryan）
- SLAM 常用：**前-左-上坐标系 + 右手系 + z-y-x 顺序 + 动轴**
- **缺点：万向锁（Gimbal Lock），不适于插值迭代**

### 角轴（旋转向量）
- 一个旋转 = 一个旋转轴 n + 旋转角 θ
- 用三维向量表示：`θ·n`（方向=轴，长度=角）
- 紧凑，但有周期性（2π 同一旋转）

### 四元数（推荐用于工程实现）
$$
q = q_0 + q_1 i + q_2 j + q_3 k \quad (i^2=j^2=k^2=ijk=-1)
$$

- 单位四元数可表达三维旋转
- 优点：**紧凑、无奇异性、可平滑插值**
- 与复数类比：自己和自己的运算像复数，自己和别人的运算像叉乘

### 三者对比

| 表达 | 自由度 | 紧凑性 | 奇异性 | 适用场景 |
|---|---|---|---|---|
| 旋转矩阵 | 9 | 冗余 | 无 | 推导、几何 |
| 欧拉角 | 3 | 紧凑 | 万向锁 | 人机交互 |
| 角轴 | 3 | 紧凑 | 2π 周期 | 几何直观 |
| 四元数 | 4 | 紧凑 | 无 | 工程实现 |

## 第8页 李群李代数与齐次变换 SE(3)

### 为什么要李群李代数？

> 在 SO(3) / SE(3) 上做**求导/优化**很困难，因为它们不是线性空间。

### 李群 ↔ 李代数 映射

| 李群 | 李代数 | 维度 |
|---|---|---|
| SO(2) | so(2) | 1 |
| SO(3) | so(3) | 3 |
| SE(2) | se(2) | 3 |
| SE(3) | se(3) | 6 |

### 指数映射
$$
\exp(\phi^\wedge) = R \in SO(3)
$$
反对称矩阵 `φ∧` 把三维向量变成 so(3) 中的元素，exp 后回到 SO(3)。

### 对数映射（逆运算）
$$
\phi = \ln(R)^\vee
$$

### 用途
- 在李代数上做无约束优化
- 用 BCH 近似处理扰动：exp(ξ)·exp(δξ) ≈ exp((Jl·δξ)∨)·exp(ξ)
- g2o / Ceres / GTSAM 都基于李代数实现位姿优化

### 连续变换的整体矩阵

**求 `p_w`（动轴下追踪 T_wb）：右乘**
$$
T_{wb_n} = T_{wb_1} T_{b_1 b_2} \cdots T_{b_{n-1} b_n}
$$

**求 `p_b`（动轴下计算 T_bw）：左乘**
$$
T_{b_n w} = T_{b_n b_{n-1}} \cdots T_{b_1 w}
$$

## 第9页 惯性里程计：问题描述

### 问题
以 IMU 0 时刻位置和姿态定义世界系，IMU 在 t 时刻输出：
- 加速度 `ãᵢ(t) = [a_x, a_y, a_z]ᵀ`
- 角速度 `ω̃ᵢ(t) = [ω_x, ω_y, ω_z]ᵀ`

采样率 f，间隔 `δt = 1/f`。问：t 时刻载体在世界系中的位姿 `R_w(t), p_w(t)`？

### 惯性导航系统的两种实现
| 类型 | 描述 |
|---|---|
| **稳定平台式** Stable Platform | 传感器安装在平台，与外部旋转隔离 → 读数为全局系 |
| **捷联式** Strapdown | 传感器刚性固连在设备 → 读数为机体系 |

> 现代 IMU 多为捷联式，需要在算法中做坐标变换。

### 测量值与真实值的关系
$$
\tilde\omega(t) = \omega(t) + b_g(t) + n_g(t) \\
\tilde a(t) = R^T(t)(a(t) - g) + b_a(t) + n_a(t)
$$
- `b_g, b_a` 测量 bias（时变、需估计）
- `n_g, n_a` 随机噪声

### INS 流程
```
Pose Update  →  R(k+1)
       ↑
Calculate linear ACC  →  去重力
       ↑
Update velocity  →  V(k+1)
       ↑
Update position  →  P(k+1)
       ↑
IMU 输入 ω̃, ã
```

## 第10页 姿态解算五种方法对比

### 五大方法

| # | 方法 | 公式核心 | 备注 |
|---|---|---|---|
| 1 | **角速度反对称矩阵法** | `R(t+δt) = (I + δt·ω∧)·R(t)` | 近似计算，最快 |
| 2 | **罗德里格斯公式** | `R = I + S·sinθ + S²(1-cosθ)` | 经典 |
| 3 | **罗德里格斯公式(简)** | `R = cosθ·I + (1-cosθ)nnᵀ + sinθ·n∧` | 视觉SLAM十四讲 |
| 4 | **指数映射** | `R(t+δt) = exp(δt·ω∧)·R(t)` | Groves |
| 5 | **四元数法** | `q(t+δt) = Ω(ω·δt)·q(t)` | 无奇异性 |

### 四元数更新细节

构造旋转向量对应的四元数：
$$
\delta q = \begin{bmatrix} \cos(\delta\theta/2) \\ \omega/||\omega|| · \sin(\delta\theta/2) \end{bmatrix}
$$
更新：
$$
q(t+δt) = q(t) ⊗ δq
$$
最后**归一化**保持单位四元数性质。

### 对比实验
- **Ekinox 高精度 IMU**（200Hz）：5 种方法 10 分钟结果几乎完全相同
- **MPU6000 手机级 IMU**（200Hz）：10 分钟就出现大偏差，必须融合外部观测

### 方法选型

| 场景 | 推荐方法 |
|---|---|
| 行人 PDR（手机/鞋戴 IMU） | 罗德里格斯 / 指数映射 |
| 视觉/雷达惯性里程计 | 罗德里格斯 / 四元数 |
| 无人机/手机（俯仰角大变化） | 四元数 |
| 有周期性外部修正的 PDR | 反对称矩阵（最快） |

## 第11页 速度位置更新与 ESKF 偏差滤波

### 速度位置更新公式

精确（连续）：
$$
v(t) = v(0) + \int_0^t R(\tau)(a(\tau) - g)\, d\tau \\
p(t) = p(0) + \int_0^t v(\tau)\, d\tau
$$

离散近似（IMU 采样）：
$$
R(t+δt) = R(t) · \exp((ω̃ - b_g)δt) \\
v(t+δt) = v(t) + g·δt + R(t)(ã - b_a)·δt \\
p(t+δt) = p(t) + v(t)·δt + \tfrac{1}{2}g·δt² + \tfrac{1}{2}R(t)(ã - b_a)·δt²
$$

> ⚠️ 双重积分使 bias 误差**快速累积**，纯 IMU 几十秒就漂出可用范围

### ESKF（Error-State Kalman Filter）

状态向量：
$$
δx = [δp, δv, δφ, δb_a, δb_g]ᵀ
$$

只追踪 INS 估计值与真实值的**偏差**，而不是绝对状态。

### ESKF 三步骤

1. **预测**（仅 IMU 输入）：
   - 运行 INS 系统
   - 状态转移矩阵 `F_k`
   - 方差传播：`P = F P Fᵀ + G Q Gᵀ`

2. **更新**（有外部观测时）：
   - 构造观测矩阵 `H_k`
   - 卡尔曼增益：`K = P Hᵀ (H P Hᵀ + R)⁻¹`
   - 状态修正：`δx = K(z - h(x))`

3. **注入 + 重置**：
   - 把偏差注入到 INS 的位置/速度/姿态
   - **偏差状态置零**（δx = 0）

### 常用外部观测

| 来源 | 方法 |
|---|---|
| 行人 PDR | **ZUPT** 零速检测（脚落地瞬间） |
| 视觉惯性 | 视觉里程计位姿 |
| 雷达惯性 | 雷达里程计位姿 |

### 实验结果对比（MPU6000 仿真）
| 设置 | 速度误差 | 位置误差 |
|---|---|---|
| bias=0 | 较小 | 较慢漂移 |
| bias=经验值 | 显著漂移 | 快速漂移 |
| bias=0 + ZUPT | 减小 | 减小（带毛刺） |
| bias=0 + 周期性视觉观测 | 显著减小 | 显著减小 |

## 第12页 相机模型：针孔·畸变·内外参

### 小孔成像模型

$$
\frac{Z}{f} = -\frac{X}{X'} = -\frac{Y}{Y'}
$$
（翻转到前面后去掉负号）

### 像素坐标变换

世界 → 相机 → 归一化平面 → 像素
$$
\begin{bmatrix} u \\ v \\ 1 \end{bmatrix} = \frac{1}{Z}
\begin{bmatrix} f_x & 0 & c_x \\ 0 & f_y & c_y \\ 0 & 0 & 1 \end{bmatrix}
\begin{bmatrix} X \\ Y \\ Z \end{bmatrix}
$$

### 内参 K
- `f_x, f_y`：焦距（像素单位）
- `c_x, c_y`：主点（图像中心）
- 一次标定终身使用

KITTI 示例：
```
P0 = [707.0912, 0, 601.8873, 0;
       0, 707.0912, 183.1106, 0;
       0, 0, 1, 0]
```

### 外参（SLAM 估计的目标！）
$$
P_{cam} = T_{cw} · P_{world}
$$
注意：**相机模型中是 T_cw，INS 追踪的是 T_wc**

### 畸变模型
- **径向畸变**（桶形/枕形）：多项式
  `r' = r(1 + k₁r² + k₂r⁴ + k₃r⁶)`
- **切向畸变**（镜头装配误差）：
  `x' = x + 2p₁xy + p₂(r² + 2x²)`

### 双目相机
$$
Z = \frac{f·b}{d}
$$
- `b`：基线，`d`：视差
- 视差 d 最小 1 像素 → 最大可测深度 `Z_max = f·b`

### RGB-D 相机
- 结构光 / ToF
- 直接给出像素深度 → 拼点云 `p_w = T_wc · K⁻¹·[u,v,1]ᵀ·Z`

## 第13页 ORB 特征：FAST 角点 + BRIEF 描述子

### 为什么用特征点？
- **可重复**：不同视角/光照都能提取
- **可区别**：不同点描述子不同
- **高效**：描述子紧凑、匹配快
- **本地**：在小区域提取

### ORB = Oriented FAST + Rotated BRIEF

#### 关键点：Oriented FAST
1. **FAST 角点检测**：
   - 像素 p 与半径 3 的圆上 16 个邻居比较
   - 连续 N 个（如 9、11、12）亮度差 > T → 是角点
   - 非极大值抑制去除扎堆

2. **改进 1：Harris 响应值排序**
   - 计算 Harris 响应 R = det(A) - k·tr(A)²
   - 选取前 N 个响应最大的

3. **改进 2：图像金字塔（尺度不变）**
   - 比例因子 `scaleFactor = 1.2`，层数 `nlevels = 8`
   - 每层都检测 FAST，总和作为图像特征

4. **改进 3：方向（旋转不变）**
   - 矩法计算质心方向 θ
   - BRIEF 描述子按 θ 旋转

#### 描述子：BRIEF
- 在关键点邻域选 N 对像素 `{p_i, q_i}`
- 比较灰度：`b_i = 1 if p_i > q_i else 0`
- 一般 N = 128/256/512 → 二进制串
- 海明距离 = 异或位计数 → 极快

### ORB 总结
```
速度：极快（无 SIFT/SURF 的多尺度 DOG）
鲁棒：尺度 + 旋转不变
实用：OpenCV `cv::ORB::create()`
```

## 第14页 2D-2D 对极几何与八点法

### 场景：单目两帧，无地图

已知：`p₁ ↔ p₂` 匹配点对
求：相机相对位姿 `T₁₂ = (R, t)`

### 几何关系
- O₁, O₂, P 三点共面 → **对极约束**
- `e₁, e₂` 极点（光心连线与像平面交点）
- `l₂` 极线（p₂ 必在 l₂ 上）

### 对极约束推导

归一化坐标 `x₁, x₂`：
$$
x_2^T E x_1 = 0
$$

**本质矩阵** `E = t∧ · R`（已知 K 时从匹配点求）

**基础矩阵** `F = K⁻ᵀ E K⁻¹`（K 未知时）

### E 的性质
- 奇异值必然是 `[σ, σ, 0]ᵀ`
- 尺度等价性（乘任意非零常数不变）→ 实际 5 个自由度
- t 和 R 各 3 个自由度 → 共 5 个独立

### 八点法

将 E 看作 9 元素向量 `e`：
$$
\begin{bmatrix} u_1²u_2 & u_1u_2v_2 & u_1u_2 & ... \\ ... \end{bmatrix} e = 0
$$
- 8 对点 → 8 个方程 → 线性解 E
- SVD 后强制奇异值为 `[σ,σ,0]`
- SVD 分解 `E = U diag(σ,σ,0) Vᵀ`
- 4 组解 `{R, ±t}`，只有深度都为正的那组

### 五点法
- 利用 E 的非线性约束
- 最少 5 对点 → 更鲁棒但实现复杂

### 单应矩阵 H（特征点共面时）

平面场景下：
$$
p_2 = H · p_1
$$
- 4 对点 → 8 个方程 → 求 H
- 适用于航拍地面、室内地面

### 实践注意
- `cv::findFundamentalMat(..., CV_FM_8POINT)`
- `cv::findEssentialMat(..., focal_length, principal_point)`
- 多于 8 对点：用 RANSAC 或最小二乘
- **尺度不确定**：恢复的 t 不是真实米

## 第15页 三角化：从像素反推 3D 坐标

### 已知
- 两帧图像的像素坐标 `p₁, p₂`
- 相机运动 `R₂₁, t₂₁`
- 内参 K

### 求
- 空间点 P 的 3D 坐标

### 几何本质
P 在两个相机光心发出的射线 `O₁p₁` 和 `O₂p₂` 的**交点**

### DLT（直接线性变换）

设 `x₁ = K⁻¹p₁`, `x₂ = K⁻¹p₂`（归一化），则有：
$$
s_1 x_1 = P, \quad s_2 x_2 = R·P + t
$$

利用叉乘消掉 `s₁, s₂`：
$$
x_1 × P = 0, \quad x_2 × (R·P + t) = 0
$$

> 两组各取两个独立方程 → 4 个方程求 4 未知数 → SVD 解齐次线性方程组

### 三角化的问题
1. **必须有平移**：纯旋转无法三角化（射线平行）
2. **大平移 → 匹配困难**
3. **相机正前方无视差** → 图像中心的点深度不确定
4. **尺度耦合**：三角化深度受 t 的尺度影响

### 工程处理
- DLT 结果作为初始值
- 后续帧改用 PnP（3D-2D），跳过 E 的尺度
- 或者用 mid-point 法：求两射线公垂线中点
- 或者直接做 BA（Bundle Adjustment）

## 第16页 3D-2D：PnP 与 Bundle Adjustment

### 场景
已知 N 个 3D 点 `{P_i}` 和它们在当前帧的 2D 投影 `{u_i}`，求相机位姿 `T = (R, t)`

> RGB-D、双目后续帧的常用方法（避免了 2D-2D 的尺度问题）

### 代数解法
| 方法 | 所需点对 | 备注 |
|---|---|---|
| **DLT** | 6 对 | 把 R,t 看作 12 个独立未知数，求后投影到 SE(3) |
| **P3P** | 3 对 | 透视三点，几何约束求 4 组解 |
| **EPnP** | 4+ 对 | 高效、利用所有点 |
| **UPnP** | 不限 | 不需要内参已知 |

### 优化解法：Bundle Adjustment（BA）

定义**重投影误差**：
$$
e_i = u_i - \frac{1}{s_i} K \exp(\xi^\wedge) P_i
$$

最小化：
$$
\min_{\xi, P} \sum_i \|e_i\|^2
$$

求解：高斯-牛顿 / Levenberg-Marquardt

### 雅可比推导

**误差对位姿 ξ 的导数**（链式法则）：
$$
\frac{\partial e}{\partial \xi} = -\frac{\partial u}{\partial P'} · \frac{\partial P'}{\partial \xi}
$$

- `∂u/∂P'`：图像投影雅可比（2×3）
- `∂P'/∂ξ`：SE(3) 李代数雅可比（3×6）

**误差对空间点 P 的导数**（3×3）

### g2o 中的实现
- `VertexSE3Expmap`：6 维李代数位姿顶点
- `VertexSBAPointXYZ`：3D 点顶点
- `EdgeProjectXYZ2UV`：二元边，连接两个顶点
- 实现 `computeError()` 和 `linearizeOplus()` 两个虚函数

## 第17页 3D-3D：ICP 与 SVD 求解

### 场景
已知两组配对的 3D 点 `{p_i}` 和 `{q_i}`，求 R, t
> RGB-D、激光 SLAM 中常用

### 误差函数
$$
\min_{R,t} \sum_i \|q_i - (R p_i + t)\|^2
$$

### SVD 求解

1. **去质心**：
   `p̄ = (1/n)Σp_i`, `q̄ = (1/n)Σq_i`
   `p_i' = p_i - p̄`, `q_i' = q_i - q̄`

2. **构造协方差矩阵**：
   `W = Σ p_i' · q_i'ᵀ`

3. **SVD**：`W = U Σ Vᵀ`

4. **最优 R**：
   `R = U Vᵀ`
   若 `det(R) < 0`，取 `R = U · diag(1,1,-1) · Vᵀ` 修正

5. **最优 t**：
   `t = q̄ - R · p̄`

### 收敛性
- ICP 是**非线性迭代**方法
- 需要初始估计（粗配准）否则陷入局部最优
- 实践中常用 K-D tree 加速最近邻搜索

### 与视觉方法的对应

| 视觉方法 | 几何关系 | 输入 | 解法 |
|---|---|---|---|
| 2D-2D | 对极几何 | 像素对 | 八点法 E/F |
| 3D-2D | 投影 | 3D + 像素 | PnP / BA |
| 3D-3D | 刚体 | 3D-3D 对 | ICP / SVD |

## 第18页 光流法：LK 稀疏光流与多层金字塔

### 动机
特征点法的痛点：
- 特征提取 ~10+ms（ORB）
- 暴力匹配 O(n²)

### 灰度不变假设
$$
I(x, y, t) = I(x+dx, y+dy, t+dt)
$$

Taylor 展开 + 保留一阶：
$$
I_x · \frac{dx}{dt} + I_y · \frac{dy}{dt} = -I_t
$$
- 一个方程两个未知数 → **欠定**

### Lucas-Kanade 光流
- 在一个 w×w 窗口内假设运动一致
- w×w 个方程 → 超定最小二乘解 (u, v)

### 多层金字塔（应对大运动）
- 构造图像金字塔（缩放 1/2 逐层）
- 从最顶层开始算光流
- 上一层的结果作为下一层的初始值
- 单层只能在初值接近最优时效果好

### 评价
- 速度快（<1ms）
- 稀疏：只能追踪少量关键点
- 灰度不变假设脆弱（曝光变化易失效）

### 应用
- LK + 对极几何 = 2D-2D VO 替代方案
- 在已知初始估计时非常有效

## 第19页 直接法：用光度误差最小化求位姿

### 直接 vs 特征点 vs 光流

| 方法 | 找对应点 | 算位姿 |
|---|---|---|
| 特征点法 | 描述子匹配（O(n²)） | E/F 或 PnP |
| 光流法 | LK（灰度） | E/F |
| **直接法** | 同时找 + 算位姿 | **光度误差最小化** |

### 直接法推导

已知初始估计 `R, t`，定义**光度误差**：
$$
e = I_2(\pi(T \cdot P)) - I_1(p_1)
$$
其中 π 是投影函数。

> 同时求解相机运动和对应点！

### 雅可比（链式法则 = 三项相乘）

$$
\frac{\partial e}{\partial \xi} = \underbrace{\frac{\partial I_2}{\partial u}}_{\text{像素梯度}} · \underbrace{\frac{\partial u}{\partial q}}_{\text{投影雅可比}} · \underbrace{\frac{\partial q}{\partial \xi}}_{\text{SE(3) 李代数}}
$$

- 第一项：图像梯度（数值差分）
- 第二项：`[-fy/Z, 0, fy·X/Z²; 0, -fx/Z, fx·Y/Z²]`
- 第三项：`[-1/Z, 0, X/Z², X·Y/Z, -(1+X²/Z²), Y/Z; 0, -1/Z, Y/Z², 1+Y²/Z², -X·Y/Z, -X/Z]`

### 三种直接法（按 P 来源）

| 类型 | P 数量 | 速度 | 适用 |
|---|---|---|---|
| **稀疏** | 数百~上千关键点 | 最快 | 实时 VO |
| **半稠密** | 梯度明显的像素 | 中等 | 折中 |
| **稠密** | 全部像素（数十万） | 需 GPU | 重构 |

### 直接法优劣势

**优势**：
- 省去特征提取（10+ms）
- 只需像素梯度，不必是角点
- 白墙等弱纹理场景仍可工作

**劣势**：
- 灰度不变假设脆弱
- 单像素区分性差
- 图像非凸 → 需要好的初值（用金字塔）

### 直接法的 g2o 实现
- 自定义一元边 `EdgeSE3ProjectDirect`
- 实现 `computeError()` 计算光度差
- 实现 `linearizeOplus()` 计算上述 1×6 雅可比

## 第20页 GraphSLAM 与回环检测（词袋模型）

### 后端为什么需要图优化？

纯前端 VO 的累积漂移：
- 5 帧漂 1cm，1000 帧就漂 2m
- 局部约束无法消除

### 图优化的目标函数
$$
\min_{x} \sum_{(i,j) \in E} \|e_{ij}(x_i, x_j)\|^2_{\Omega_{ij}}
$$
- `x_i`：位姿或路标的估计
- `e_{ij}`：边的误差（预测 vs 观测）
- `Ω_{ij}`：信息矩阵（观测精度的逆）

### 边的类型

| 边类型 | 含义 |
|---|---|
| **里程计边** | 相邻帧相对位姿 |
| **观测边** | 帧-路标观测（距离 / 角度 / 距离+角度） |
| **回环边** | 远距离帧间约束 |

### 高斯-牛顿求解

线性化：`F(x + Δx) ≈ e + J·Δx`
求解正规方程：`H·Δx = -b`，其中 `H = Jᵀ J, b = -Jᵀ e`

利用 H 的稀疏性（0.17% 非零）→ Cholesky 分解高效求解

### 改进算法

| 算法 | 解决什么 |
|---|---|
| **Levenberg** | 步长过大 |
| **Marquardt** | 步长过大（更鲁棒） |
| **阻尼牛顿** | 凹区发散 |

### 开源 SLAM 后端库

| 库 | 语言 | Stars | 特点 |
|---|---|---|---|
| Ceres | C++ | 1.7K | Google 大规模非线性优化 |
| G2O | C++/Matlab/Python | 1.7K | 通用图优化 |
| GTSAM | C++ + wrapper | 725 | Georgia Tech 因式图 |
| miniSAM | Python | 224 | 轻量级 |

### 回环检测的目的

- 给出远距离约束 → 全局一致性
- 消除累积漂移

### 回环检测的三种方法

1. **基于特征匹配**（传统）
2. **基于机器学习（CNN/RF）** 预测 2D-3D 匹配
3. **基于深度学习直接回归 6DoF 位姿**（PoseNet 等）

### 词袋模型 BoW（Bag of Words）

| 概念 | 解释 |
|---|---|
| 单词 | 离线聚类得到的一组相似描述子 |
| 词典 | 所有单词的集合（K-means 或层次聚类） |
| 词向量 | 图像中各单词是否出现（0/1 + TF-IDF 权重） |
| 相似度 | 词向量间的 L1 范数 |

### BoW 回环检测流程

```
离线：训练图像 → K-means/层次聚类 → BoW 字典
在线：
  1. 当前帧 → 特征 → 查字典 → 词向量
  2. 在 Image Database 中查最近邻（逆序索引加速）
  3. 相对前一帧得分归一化（防误报）
  4. 时间一致性（组匹配）
  5. 结构一致性（特征点几何验证）
```

### 开源词袋库

| 库 | 描述 |
|---|---|
| DBoW | 原始 C++ 实现 |
| DBoW2 | 正/逆序索引 |
| DBoW3 | DBoW2 改进 |
| FBOW | 比 DBoW2 快 80 倍 |

### 视觉重定位（已知地图下的相机定位）

应用：AR 导航、VR 交互、机器人回到已知环境

主流方法：
- **图像检索**（DBoW / VLAD / NetVLAD）
- **2D-3D 匹配**（暴力 / KNN / SuperGlue / LoFTR）
- **位姿解算**（对极几何 + RANSAC PnP）

### 整个 SLAM 系统闭环

```
前端 VO ──→ 局部位姿 ─┐
                      ├→ 后端图优化 ─→ 全局一致位姿
回环检测 ──→ 远约束 ─┘      ↑
                          └── 视觉重定位回环节点
```
