1.6 访存模型详解

第1章　并行计算机结构模型

1.6.1 UMA（Uniform Memory Access）均匀存储访问

  • 物理存储器被所有处理器均匀共享。
  • 所有处理器访问任何存储字取相同的时间。
  • 每台处理器可带私有高速缓存。
  • 外围设备也可以一定形式共享。
                    P1   P2   …   Pn
                     │    │        │
                     └────┴────────┴─── 系统互连（总线 / 交叉开关 / 多级网络）──►
                                                              │
                                                          ┌───┴────┐
                                                          │  I/O   │
                                                          │ SM1...SMm│
                                                          │ 共享存储│
                                                          └────────┘

1.6.2 NUMA（Nonuniform Memory Access）非均匀存储访问

  • 共享存储器物理上分布在所有处理器中，所有本地存储器的集合组成全局地址空间。
  • 处理器访问存储器的时间不一样：访问本地存储器 LM 或群内共享存储器 CSM 较快，而访问外地存储器或全局共享存储器 GSM 较慢。
  • 每台处理器可带私有高速缓存，外设可以某种形式共享。

1.6.3 COMA（Cache-Only Memory Access）全高速缓存存储访问

  • 各处理器节点中没有存储层次结构，全部高速缓存组成了全局地址空间。
  • 利用分布的高速缓存目录 D 进行远程高速缓存的访问。
  • COMA 中的高速缓存容量一般都大于 2 级高速缓存容量。
  • 使用 COMA 时，数据开始时可任意分配，运行时最终会被迁移到要用到它的地方。

1.6.4 CC-NUMA（Coherent-Cache NUMA）高速缓存一致性非均匀存储访问

  • 大多数使用基于目录的高速缓存一致性协议。
  • 保留 SMP 结构易于编程的优点，也改善常规 SMP 的可扩放性。
  • 实际上是一个分布共享存储的 DSM 多处理机系统。
  • 显著优点：程序员无需明确地在节点上分配数据，硬件和软件自动迁移数据。
  I/O  NIC, DIR, RC                            I/O  NIC, DIR, RC
  ─┬───┬─────┐                                ─┬───┬─────┐
   │Mem│P/C  │          系统互连网络             │Mem│P/C  │
   └───┴─────┘ ◄──────────────────────────────►  └───┴─────┘
   节点 1                                  节点 N
       (总线或交叉开关)                       (总线或交叉开关)

1.6.5 NORMA（No-Remote Memory Access）非远程存储访问

  • 所有存储器是私有的。
  • 绝大多数 NUMA 都不支持远程存储器的访问。
  • 在 DSM 中，NORMA 就消失了。

1.8 对称多处理机 SMP

第1章　并行计算机结构模型

  • SMP：采用商用微处理器，通常有片上和片外 Cache，基于总线连接，集中式共享存储，UMA 结构。
  • 例子：SGI Power Challenge、DEC Alpha Server、曙光 1 号。

优点

  • 对称性
  • 单地址空间，易编程性，动态负载平衡，无需显示数据分配
  • 高速缓存及其一致性，数据局部性，硬件维持一致性
  • 低通信延迟，Load/Store 完成

问题

  • 欠可靠（BUS、OS、SM）
  • 通信延迟相对于 CPU 上升，竞争加剧
  • 慢速增加的带宽（MB 每 3 年翻倍，IOB 更慢）
  • 不可扩放性 → CC-NUMA
第1章

1.9 大规模并行机 MPP

第1章　并行计算机结构模型

  • 由成百上千个处理器组成的大规模计算机系统。
  • NORMA 结构，高带宽低延迟定制互连。
  • 可扩放性：Mem、I/O 平衡设计。
  • 通用性和可用性：PVM、MPI、交互、批处理、互连对用户透明、单一系统映像、故障处理。
  • 例子：Intel Option Red、IBM SP2、曙光 1000。
第1章

1.10 机群型大规模并行机 SP2

第1章　并行计算机结构模型

  • 设计策略：机群体系结构、标准环境、标准编程模型、系统可用性、精选的单一系统映像。
  • 系统结构：HPS（高性能开关）多级 Ω 网络；宽节点、窄节点和窄节点 2。
第1章

1.11 工作站机群 COW

第1章　并行计算机结构模型

  • 特征：分布式存储，MIMD，工作站 + 商用互连网络，每个节点是完整计算机。
  • 优点：投资风险小、系统结构灵活、性能/价格比高、能充分利用分散资源、可扩放性好。
  • 问题：通信性能、并行编程环境。
  • 例子：Berkeley NOW、Alpha Farm、FXCOW。

2.2 网络性能指标

第2章　并行计算机互连与通信

  • 节点度（Node Degree）：射入或射出一个节点的边数。在单向网络中，入射和出射边之和称为节点度。
  • 网络直径（Network Diameter）：网络中任何两个节点之间的最长距离（最大路径数）。
  • 对剖宽度（Bisection Width）：对分网络各半所必须移去的最少边数（允许两边差 1 行）。
  • 对剖带宽（Bisection Bandwidth）：每秒钟内，在最小对剖平面上通过所有连线的最大信息位（或字节）数。
  • 对称性（Symmetry）：从任一节点观看网络都一样。
第2章

2.3 静态互连网络

第2章　并行计算机互连与通信

  • 一维线性阵列（1-D Linear Array）：节点度 2（内节点），直径 N−1，对剖宽度 1。
  • 环形（Ring）：首尾相连，节点度恒为 2，直径 ⌊N/2⌋（双向）或 N−1（单向），对剖宽度 2。
  • 二维网孔（2-D Mesh）：节点度 4，直径 2(N−1)，对剖宽度 N。
  • Illiac 网孔：垂直环绕、水平蛇状，节点度恒为 4，直径 (√N−1)，对剖宽度 √N。
  • 2-D 环绕（2-D Torus）：垂直水平均环绕，节点度恒为 4，直径 2⌊√N/2⌋，对剖宽度 2√N。
  • 二叉树：节点度 3，对剖宽度 1，直径 2(⌈log₂N⌉−1)。
  • 星形网络：节点度 N−1，直径 2，对剖宽度 ⌊N/2⌋。
  • 二叉胖树（Fat Tree）：通路自叶向根逐渐变宽，解决根瓶颈问题。
  • 超立方（n-Cube）：N=2ⁿ 个顶点，节点度 n，直径 n，对剖宽度 N/2。
  • 立方环：3-立方的每个顶点代之以一个环，每个顶点度为 3。

2.5 嵌入（Embedding）

第2章　并行计算机互连与通信

  • 将网络中的各节点映射到另一个网络中去。
  • 用膨胀（Dilation）系数来描述嵌入质量：被嵌入网络中的一条链路在所要嵌入的网络中对应所需的最大链路数。
  • 如果膨胀系数为 1，称为完美嵌入。
  • 环网可完美嵌入到 2-D 环绕网中
  • 超立方网可完美嵌入到 2-D 环绕网中
第2章

2.6 动态互连网络

第2章　并行计算机互连与通信

2.6.1 总线（PCI, VME, Multibus, Sbus, MicroChannel）

多处理机总线系统的主要问题：

  • 总线仲裁、中断处理、协议转换
  • 快速同步、高速缓存一致性协议
  • 分事务、总线桥和层次总线扩展

2.6.2 交叉开关（Crossbar）

  • 单级交换网络，可为每个端口提供更高的带宽。
  • 象电话交换机一样，交叉点开关可由程序控制动态设置其处于"开"或"关"状态。
  • 两种使用方式：
  • 用于对称多处理机或多计算机机群中处理器间通信
  • 用于 SMP 服务器或向量超级计算机中处理器和存储器之间的存取

2.6.3 交换开关模块

  • n 个输入和 n 个输出，每个输入可连接到任意输出端口。
  • 只允许一对一或一对多的映射，不允许多对一（会冲突）。

2.6.4 多级互连网络 MIN（Multistage Interconnection Network）

  • 单级交叉开关级联起来形成 MIN。
  • 级间互连：均匀洗牌、蝶网、多路均匀洗牌、交叉开关、立方连接。
  • n 输入的 Ω 网络需要 log₂n 级 × 2 列开关。
  • Cedar 多处理机系统（Illinois 大学）采用了 Ω 网络。
  • Cray Y/MP 多级网络：支持 8 个向量处理器和 256 个存储器模块间的数据传输，能避免 8 个处理器同时存取冲突。
第2章

2.8 标准互连网络

第2章　并行计算机互连与通信

2.8.1 Myrinet

  • 厂商：Myricom 公司设计；基于 Caltech 多计算机和 VLSI 技术和 USC 的 ATOMIC/LAN 技术。
  • 假设任意拓扑结构，不必限定为开关网孔或任何规则结构。
  • 数据链路层具有可变长的包格式，对每条链路施行流控制和错误控制，使用切通选路法及定制的可编程主机接口。
  • 物理层：全双工 SAN 链路，最长 3 米，峰值速率 (1.28+1.28) Gbps（目前有 2.56+2.56）。
  • Myrinet 交换开关：8, 12, 16 端口。
  • Myrinet 主机接口：32 位的称作 LANai 芯片的用户定制的 VLSI 处理器。
  • 2002 年 11 月 TOP500 中 140 台使用 Myrinet，包括前 100 中的 15 台。

2.8.2 高性能并行接口 HiPPI

  • Los Alamos 国家实验室 1987 年提出，统一不同产商大型机和超级计算机的接口。
  • 1993 年 ANSI X3T9.3 委员会认可了 HiPPI 标准，覆盖物理和数据链路层。
  • 单工的点到点数据传输接口，速率 800 Mbps 到 1.6 Gbps。
  • 超级 HiPPI：潜在 6.4 Gbps 速率（比 HiPPI 快 8 倍且时延低）。
  • SGI 和 Los Alamos 开发了速率高达 25.6 Gbps 的 HiPPI 交换开关。

2.8.3 光纤通道 FC（Fiber Channel）

  • 通道和网络标准的集成；既可共享介质，也可交换技术。
  • 速度范围：100 / 133 / 200 / 400 / 800 Mbps，未来 1 / 2 / 4 Gbps。
  • 支持点到点、仲裁环及交换光纤连接。

2.8.4 FDDI（Fiber Distributed Data Interface）

  • 光纤分布式数据接口。
  • 双向光纤令牌环提供 100–200 Mbps 数据传输速率。
  • 具有互连大量设备的能力。
  • 传统 FDDI 仅以异步方式操作。

2.8.5 ATM（Asynchronous Transfer Mode）

  • ATM 论坛（1991）和 ITU 标准定义。
  • 独立于介质的消息传输协议，将消息段变成更短的固定长度 53 字节的报元进行传输。
  • 基于报元交换机制。
  • 网络支持 25 / 51 / 155 / 622 Mbps 速率。

2.9.1 基本术语

  • 消息（Message）：在多计算机系统处理节点之间传递包含数据和同步信息的信息包。
  • 包（Packet）：长度随协议不同而不同（64–512 位），是信息传送的最小单位。
  • 片（Flit）：长度固定，一般为 8 位。

2.9.2 选路模式分类

按传输模式分：

模式说明
一到一（单播）一条信包从源到唯一目的地
多到一（集中）多个处理器发送到同一目的地；每条信包有且仅有一个目的地
一到多（多播）一条信包到一组目的地
一到所有（广播、组播）一条信包到所有处理器（每节点最多一份）
多到多（会议）每个处理器都向所有其他处理器发消息

按路径计算分：

  • 静态选路：事先算好传输路径（offline）
  • 动态选路：没有事先计算好的路径（online）

按交换方式分：

  • 线路交换
  • 存储-转发（信包）
  • 虫孔（Wormhole）

2.9.3 选路算法的三种机制

  • 基于算术的：开关中具有简单的算术运算功能，如维序选路。
  • 基于源地址的：在源点时就将沿路径的各个开关的输出端口地址 p₀,p₁,…,pₙ 包在信包头部，每个开关只对信包头的输出端口地址进行剥离。
  • 基于查表的：开关中含有选路表，对信包头中的选路域查出输出端口地址。

2.9.4 选路分类

  • 最短路径/非最短路径（贪心选路/随机选路）：维序选路是贪心的，二阶段维序选路是随机的。
  • 确定选路/自适应选路（寻径确定/寻径视网络状况）。
  • 维序选路（Dimension-Ordered Routing）：一种确定的最短路径选路。
  • 二维网孔中的维序选路：X-Y 选路
  • 超立方中的维序选路：E-立方选路

2.9.5 X-Y 选路算法

begin
  step1: 沿 X 方向将信包送至目的地处理器所在的列
  step2: 沿 Y 方向将信包送至目的地处理器所在的行
end

示意：源 (2,1) → 目的 (7,6)，先沿 X 方向、再沿 Y 方向移动。

2.9.6 E-立方选路算法

路由计算：

s_{n-1} s_{n-2} … s_1 s_0   (源地址)
                          ⊕
d_{n-1} d_{n-2} … d_1 d_0   (目的地址)
                          =
r_{n-1} r_{n-2} … r_1 r_0   (路由值)

路由过程：依次访问 r_i=1 的维。

示例：源 S=0110，目的 D=1101，路由值 R=1011，路径：

0110 → 0111 → 0101 → 1101

2.10 开关技术

第2章　并行计算机互连与通信

2.10.1 信道与时延参数

  • 信道带宽 b：每个信道有 w 位宽和信号传输率 f = 1/t，b = w·f bits/sec。
  • 节点和开关的度：与节点和开关相连的信道数目。
  • 路径：信包在网络中走过的开关和链路（link）序列。
  • 路由长度或距离：路径中包括的链路数。

信包传输性能参数：

  • 启动时间 t_s：准备包头信息等
  • 节点延迟时间 t_h：包头穿越相邻节点的时间
  • 字传输时间 t_w：传输每个字的时间
  • 链路数 l、信包大小 m

2.10.2 存储-转发（Store-and-Forward）选路

  • 消息被分成信包，每个信包都含有寻径信息。
  • 当一个信包到达中间节点 A 时，A 把整个信包放入通信缓冲器中，然后选路到下一个相邻节点 B。
  • 传输时间：

$$t_{comm}(SF) = t_s + (mt_w + t_h) \cdot l = O(ml)$$

  • 缺点：每个节点必须缓冲整个消息，缓冲器较大；网络时延与节点数成正比。

2.10.3 切通（Cut-Through）选路

  • 在传递一个消息之前，就为它建立一条从源节点到目的节点的物理通道。
  • 传输时间：

$$t_{comm}(CT) = t_s + mt_w + lt_h = O(m+l)$$

  • 缺点：物理通道非共享；传输过程中物理通道一直被占用。

2.10.4 虫孔（Wormhole）选路

  • 由 Dally 于 1986 年提出。
  • 把消息分成许多很小的片（flit）：
  • 头片：包含所有寻径信息
  • 数据片：中间数据
  • 尾片：包含消息结束符
  • 每个节点上只需缓冲一个片。
  • 用头片直接牵引一条从输入链路到输出链路的路径。每个消息中的片以流水方式在网络中向前"蠕动"。

优点：

  1. 每个节点缓冲器需求量小，易于 VLSI 实现。
  2. 较低的网络传输延迟（消息包较长时距离影响小）。
  3. 通道共享性好、利用率高。
  4. 易于实现 Multicast 和 Broadcast。

缺点：节点缓冲器需求量小是优点，但头片阻塞会导致后续所有片停滞。

2.10.5 距离 l 的计算

对于 p 个处理器：

网络距离 l
一维环形l ≤ ⌊p/2⌋
带环绕 Meshl ≤ ⌊p^(1/2) / 2⌋
超立方l ≤ log p

单一信包传输时间：

网络SF 模式CT 模式
一维环形t_comm(SF) = (t_s + t_w·m) + ⌊p/2⌋·t_ht_comm(CT) = t_s + mt_w + lt_h
带环绕 Mesht_comm(SF) = (t_s + t_w·m) + ⌊p^(1/2)/2⌋·t_h同上
超立方t_comm(SF) = (t_s + t_w·m) + log p · t_h同上

当 m >> p 时：t_comm(SF) ≈ t_comm(CT) = t_s + mt_w（t_s >> t_h）。

2.11 集体通信（Broadcast & All-to-All）

第2章　并行计算机互连与通信

2.11.1 一到多播送 SF 模式

环：

  1. 先左右邻近传送
  2. 再左右二个方向同时播送

时间：t_all-to-one(SF) = (t_s + t_w·m)·⌈log p⌉

环绕网孔：

  1. 先完成一行中的播送
  2. 再同时进行各列的播送（4 步）

时间：t_all-to-one(SF) = (t_s + t_w·m)·⌈2·log(√p)⌉ ≈ 2(t_s + t_w·m)·log(√p)

超立方：从低维到高维依次播送。

时间：t_all-to-one(SF) = (t_s + t_w·m)·log p

2.11.2 一到多播送 CT 模式

环：

  1. 发送至 p/2 远的处理器
  2. 同时发送至 p/4 远的处理器
  3. ……

时间：t_all-to-one(CT) = t_s·log p + t_w·m·log p + t_h·(p/2)

（t_h·p/2 较大，可忽略时简化为 (t_s + t_w·m)·log p）

网孔：先进行行播送，再同时进行列播送。

时间：t_all-to-one(CT) = 2·(t_s·log √p + t_w·m·log √p + t_h·(√p−1)) = (t_s + t_w·m)·log p + 2t_h·(√p−1)

超立方：依次从低维到高维播送。

时间：t_all-to-one(CT) = (t_s + t_w·m)·log p

2.11.3 多到多播送 SF 模式

环：同时向右（或左）播送刚接收到的信包。

时间：t_all-to-all(SF) = (p−1)·(t_s + t_w·m)

环绕网孔：

  1. 先进行行的播送
  2. 再进行列的播送

时间：t_all-to-all(SF) = 2·(p−1)·(t_s + t_w·m)

超立方：依次按维进行多到多的播送。

时间：t_all-to-all(SF) = (t_s + t_w·m)·⌈log p⌉·(p/2)

2.11.4 多到多播送 CT 模式

使用一到多的策略会造成链路竞争，CT 模式一般不直接用于多到多播送。

3.3 并行与通信开销

第3章　并行计算性能评测

并行和通信开销相对于计算可能很大。

  • PowerPC：每周期 15 ns 执行 4 flops；
  • 创建进程 1.4 ms（可执行 372000 flops）。

乒乓方法（Ping-Pong Scheme）：测量点到点通信开销。

节点 0 发送 m 个字节给节点 1；节点 1 从节点 0 接收 m 个字节后，立即将消息发回节点 0。总时间除以 2 即得到点到点通信时间。

节点 0 ──────m 字节─────► 节点 1
   ◄────────m 字节────────┘

伪代码：

if (my_node_id == 0) {                /* 发送者 */
    start_time = second();
    send an m-byte message to node 1;
    receive an m-byte message from node 1;
    end_time = second();
    total_time = end_time - start_time;
    communication_time[i] = total_time / 2;
}
else if (my_node_id == 1) {            /* 接收者 */
    receive an m-byte message from node 0;
    send an m-byte message to node 0;
}

可一般化为热土豆法（Hot-Potato）或救火队法（Fire-Brigade）：0 → 1 → 2 → … → n-1 → 0。
