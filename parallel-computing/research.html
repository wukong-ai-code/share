8. 推荐学习路线

📌 前置基础

在开始高级并行计算之前确认已掌握：

计算机体系结构 C/C++ 编程 操作系统基础 数据结构与算法

具体：缓存层次、流水线、SIMD、进程/线程、虚拟内存、上下文切换

Phase 1: 基础并行 (4–6 周)

1📖 教材：Pacheco — An Introduction to Parallel Programming
2🎥 视频：MIT 6.172 (前半) 或 Stanford CME 213
3💻 练习：OpenMP 教程 + Hybrid OpenMP/MPI 混合编程练习
4🎯 掌握：共享内存并行、消息传递、Amdahl's Law、加速比分析

Phase 2: GPU 并行 + 深度性能 (6–8 周)

1📖 教材：Kirk & Hwu — PMPP (第4版)
2🎥 视频：Stanford CS149 Lecture 7–10 (GPU + CUDA)
3💻 练习：CS149 asst3 (CUDA 渲染器) + PMPP 章节习题
4🎯 掌握：CUDA 编程模型、内存层次优化、warp/block 调度

Phase 3: 进阶专题 (8+ 周)

1📖 教材：Herlihy & Shavit — The Art of Multiprocessor Programming
2🎥 视频：Stanford CS149 后半 (内存一致性、事务内存)
3💻 练习：CS149 asst2 (任务图调度) + asst4 (Spark)
4🎯 掌握：lock-free 数据结构、内存一致性模型、分布式数据并行
💡 学习建议：以 CS149 为主线，跟着写代码（只看不写等于没学），善用 Intel VTune / NVIDIA Nsight / perf 等性能分析器可视化瓶颈。30% 理论 + 70% 动手是最佳配比。

9. 信号解读

🔵 强信号

1. GPU 是现在，AI 加速器是未来。 NVIDIA DLI 课程爆炸式增长、CUDA 生态持续扩展，GPU 并行编程已成必备技能而非可选项。
2. CS149 是"一门课通关"。 课程横跨 CPU 多核 → GPU → 分布式，理论和实践完美结合。全球学习者一致推荐为最佳入门+进阶路线。
3. 中文 HPC 社区正在成熟。 heptagonhust 的 HPC 学习路线、csdiy.wiki 等高质量中文资源，让中文学习者无需完全依赖英文渠道。
4. PMPP 第4版 + 习题全解 = 自学神器。 Kirk & Hwu 教材有完整 GitHub 习题解答，自学 CUDA 不再需要课堂环境。

🟠 注意事项

5. 并行计算"理论 vs 实践"鸿沟大。 纸上谈兵容易，实际写出 scalable 的并行程序需要大量调优经验。建议 30% 理论 + 70% 动手。
6. 中文课程偏理论。 爱课程等国内平台以讲授并行算法为主，缺少 CS149 级别的动手作业。建议以国外课程为主，中文资料作辅助。
7. Rust 并行生态值得关注。 虽然当前主流教材仍以 C++/CUDA 为主，但 Rust 的内存安全模型在并行编程中有天然优势。

🔴 需警惕

8. CUDA 锁定风险。 NVIDIA 生态强大也意味着厂商锁定。建议同时了解 OpenCL/SYCL/ROCm 等开放标准。
9. "AI 自动并行化"的炒作。 虽然 LLM 开始被用于代码并行化辅助，但复杂场景的手动优化仍不可替代。不要指望 AI 替你写高性能 CUDA kernel。
